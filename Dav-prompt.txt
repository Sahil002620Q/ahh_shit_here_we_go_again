------------------------------------------------------------------------------------------------------------------------------------------------------------------------# CampusCore Admin Panel - Production Build Prompt

## Overview
Build a production-grade, fully-functional admin dashboard for CampusCore, a comprehensive campus management app. The admin panel should provide complete control over all student-facing features with real-time management capabilities, detailed analytics, and extensive history logging.

**Design Reference**: Match the CampusCore demo aesthetic - frosted glass iOS-style design, smooth animations, dark/light mode support, minimal UI with clean glass aesthetics, and polished micro-interactions.

---

## Core Features Required

### 1. **Cafeteria Management**
- **Menu Management**
  - Add new menu items with: name, description, price, category (breakfast, lunch, dinner, snacks)
  - Edit existing menu items (name, description, price, availability status)
  - Delete menu items with confirmation modal
  - Set item availability (available/unavailable toggle)
  - Schedule menu items for specific days/dates
  - Bulk upload menu via CSV/JSON
  - View all menu items in a sortable, filterable table
  - See which items are most ordered (analytics)
  - Price history tracking

- **Daily Schedule**
  - Set breakfast, lunch, dinner timings
  - Manage special meal days
  - View upcoming week's menu overview
  - Historical menu records

---

### 2. **Notification & Announcement System**
- **Push Notifications**
  - Create notifications with: title, description, category (notice, urgent, event, holiday)
  - Schedule notifications (immediate or scheduled for specific date/time)
  - Target audience selection (all students, specific batch, specific department)
  - Set notification expiry time
  - Rich text editor for notifications
  - Add images/attachments to notifications
  - Track notification status (sent, pending, failed, viewed count)
  - Resend failed notifications
  - View notification delivery analytics

- **Notice Board Management**
  - Create/edit/delete notices
  - Set notice visibility (public, admin-only, specific groups)
  - Pin important notices to top
  - Set notice expiry date/time
  - Add attachments (PDF, images, documents)
  - Full edit history of each notice
  - View read statistics per notice
  - Search and filter notices by date, category, keyword

- **Holiday & Event Management**
  - Add holiday dates with description
  - Mark as national/college/departmental holiday
  - Set holiday notifications (auto-send announcements)
  - View holiday calendar
  - Edit/delete holidays

---

### 3. **Anonymous Chat Management**
- **Chat Control Features**
  - Enable/disable anonymous chat globally
  - Temporarily block chat for specific duration (set hours:minutes)
  - View block history with reasons
  - Set block countdown timer (show when chat will be enabled)
  - Auto-resume chat after set time

- **Comment/Chat Moderation**
  - View all anonymous chat messages in real-time
  - Search messages by keyword
  - Delete inappropriate comments with reason
  - Flag message as spam/abusive
  - Ban users from anonymous chat (set duration)
  - View deleted message history
  - Mass delete messages by date range
  - See message statistics (total, today, this week)

- **User Blocking**
  - Block specific users from anonymous chat
  - Set block duration (temporary or permanent)
  - View list of blocked users
  - Unblock users
  - View block reason history
  - See blocked user activity logs

---

### 4. **Lost & Found Management**
- **Item Management**
  - Create lost/found item posts with: title, description, category, location, date
  - Upload images of items
  - Edit post details
  - Mark as "resolved/found" or "resolved/returned"
  - Delete posts with confirmation
  - View all items in grid/list view

- **Analytics**
  - Most commonly lost items
  - Most recovered items
  - Locations with most lost items
  - Active vs resolved items count

---

### 5. **Notice Board (General)**
- **Post Management**
  - Create new board posts
  - Edit/delete posts
  - Pin important posts
  - Set expiry dates
  - Add rich media (images, videos, documents)
  - View post statistics (views, engagement)

---

### 6. **Quick Links Management**
- Add/edit/remove quick college links
- Organize links by category (academic, support, resources)
- Set link priority/order
- Track link click statistics

---

### 7. **System Analytics & History**
- **Dashboard Overview**
  - Total users/students
  - Active users today/week/month
  - Most used features
  - System health status
  - Notifications sent (today, week, month)
  - Chat activity (messages, active users)

- **Detailed History Logs**
  - Complete action history (all admin actions with timestamp)
  - User activity logs (login, feature usage)
  - Chat message archive (deleted and active)
  - Notice edit history (view all versions)
  - Menu update history
  - Login attempts (failed/successful)
  - Export history as PDF/CSV

- **Per-ID History**
  - Student activity tracking (by ID)
  - Chat participation history (anonymous - anonymous user ID)
  - Notice interaction history (viewed, not viewed)
  - Feature access logs

---

## UI/UX Requirements

### **Design Aesthetic** (Match CampusCore Demo)
- Frosted glass iOS-style cards with subtle shadows
- Smooth fade, slide, and stagger animations
- Dark theme as default with light mode toggle
- Minimal, clean UI with generous whitespace
- Rounded corners (16px+ for cards)
- Subtle gradient overlays on cards
- Smooth transitions between tabs/sections (300-500ms)
- Loading states with skeleton screens
- Toast notifications for actions (success, error, warning)
- Modal confirmations for destructive actions

### **Layout Structure**
```
Sidebar Navigation:
├── Dashboard (Overview & Analytics)
├── Cafeteria Menu
├── Notifications & Announcements
│   ├── Push Notifications
│   ├── Notice Board
│   └── Holidays
├── Chat Moderation
│   ├── Chat Control
│   ├── Message Moderation
│   └── User Blocking
├── Lost & Found
├── Quick Links
└── System Logs & History
```

### **Key UI Elements**
- **Cards**: Frosted glass effect, smooth shadows, hover lift effect
- **Buttons**: Minimal design, smooth hover/active states, icon + text combinations
- **Inputs**: Clean text fields with floating labels, date/time pickers
- **Tables**: Sortable columns, filterable rows, pagination, row actions (edit, delete)
- **Modals**: Full-screen or centered, backdrop blur, smooth open/close animations
- **Toggles**: iOS-style switches with smooth animation
- **Timeline**: For history logs, showing chronological events
- **Badges**: For status indicators (active, blocked, pending, etc.)
- **Charts**: For analytics (bar charts, line charts, pie charts)

### **Interactive Elements** (All Must Be Fully Functional)
- Every button must have click handlers
- Every toggle must switch states
- Every form must submit and validate
- Every table must sort, filter, and paginate
- Every modal must open/close smoothly
- Every delete action must show confirmation
- Every input must accept and store values
- Real-time updates where applicable

## Notes for Developer

1. **Use localStorage** to simulate backend API - this makes it feel real while being fully functional offline
2. **Implement realistic delays** (200-500ms) on data operations to simulate API calls
3. **Every interaction should have feedback** - toast notifications, loading states, success confirmations
4. **History logging is critical** - automatically log every admin action with timestamp
5. **Make it beautiful** - this is the first thing college admins will see, make them impressed
6. **Think about edge cases** - empty states, loading states, error states, success states
7. **Mobile first** doesn't mean small-only, design for desktop but ensure it works on mobile too
8. **Use the CampusCore aesthetic** as reference - frosted glass, smooth animations, minimal design
